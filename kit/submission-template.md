# Project submission kit

For onboarding ZAO projects (and outside organizers) onto Artizen.
Full context: ../research/849-zao-artizen-execution-build-plan/README.md

## Confirmed Artifact specs (build to these)

- Image (required): square 1:1, min 1000x1000px, .jpg/.png, <10MB, NO text/logo overlays.
- GIF (optional): square, min 500px, <50MB, loopable.
- Video (optional): square, <=45s, <100MB, .mp4, loopable, no overlays.
- The Artifact is the creative essence of the project, not a marketing flyer. Video/GIF outperform static in voting.
- Price fixed at $10 open-edition; 100% to creator; creator needs an Ethereum wallet for payout (sales + match + prize, ~2-4 weeks after season close).

## Submission template (fill one per project at artizen.fund/submit)

```
Project title: [name]
One-liner: [what it is in one sentence]
Description (100-300 words): [what you're making, why it matters, what's done so far -
  hit Weird + Wonderful + Working: original angle, emotional pull, signs of life/traction]
Category: [music / community / festival / tooling / etc.]
Team: [creator name(s) + roles]
Links: [site, Farcaster, X, demo]
Artifact asset: [the square image/video per specs]
Payout wallet: [ETH address]
Funds to target: ZAO Fund for Emerging Culture + [ZAO Festivals Fund / Greenpill / Funding the Commons]
```

## How a project joins a specific fund

Submit to the season -> Artizen matches it to funds whose eligibility it meets (+ director judgment)
-> community vote curates the top ~30% into competition. No separate "apply to fund X" button. To fill
the ZAO Festivals Fund: define eligibility crisply, recruit organizers to submit, rally the vote.
